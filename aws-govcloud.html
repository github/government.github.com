What is GovCloud?

AWS GovCloud (US) Region is an isolated Amazon Web Services environment used by US government agencies at the federal, state, and local levels, along with contractors, researchers, educational institutions, and other US customers.

Compliance with federal standards

In terms of boxes checked, GovCloud has received the following certifications:

{% octicon check width:18 class:"fill-green" aria-label:check %}

An Agency Authorization to Operate (ATO) from the US Department of Health and Human Services (HHS) utilizing a FedRAMP accredited Third Party Assessment Organization (3PAO)

{% octicon check width:18 class:"fill-green" aria-label:check %}

Federal Risk and Authorization Management Program (FedRAMP) at the Moderate impact level

{% octicon check width:18 class:"fill-green" aria-label:check %}

U.S. International Traffic in Arms Regulations (ITAR)

{% octicon check width:18 class:"fill-green" aria-label:check %}

The Federal Information Processing Standard (FIPS) Publication 140-2

{% octicon check width:18 class:"fill-green" aria-label:check %}

Federal Information Security Management Act (FISMA, NIST 800-37)

{% octicon check width:18 class:"fill-green" aria-label:check %}

DoD Information Assurance Certification and Accreditation Process (DIACAP), and

{% octicon check width:18 class:"fill-green" aria-label:check %}

Department of Defense (DoD) Cloud Security Model (CSM) Levels 3-5

For more information on AWS's compliance with federal and other standards, see AWS's compliance page.

Installing GitHub Enterprise on AWS GovCloud

For installation purposes, GovCloud is treated like any other AWS region. Simply follow the steps for Installing GitHub Enterprise on AWS, being sure to select the 'us-gov-west-1' specific AMI ID.

Migrating GitHub Enterprise to AWS GovCloud

Customers that are already using GitHub Enterprise can migrate from other GitHub Enterprise deployment platforms to GovCloud by following the instructions in our Admin Guide.
